0.序言

近期学习内容总结下来可以分为以下几个方面：

  • 网站开发：前端：Html+CSS+JavaScript 后端：PHP
  • HTTP协议
  • 信息收集：
    • 域名信息收集
    • DNS信息收集

1. 网站开发

1.1.基础流程简介

  1. 首先使用html建立网站的基本结构框架
  2. 接着使用CSS来对网页进行渲染美化
  3. 然后使用JavaScript实现网页基本的交互功能
  4. 使用PHP来连接网站的数据库

1.2.知识总结

1.2.5 习题：云登录系统实现总结

首先在上次作业，作业2的基础上进行修改，完善了注册以及忘记密码界面。同时，新建登陆后跳转到的个人中心界面。

首先进行的功能实现是注册功能，在这一部分，增加了邮箱信息的填写，实现了动态判断设置密码强度的功能。（保留优化点：发送邮件验证邮箱）

后台数据数据库管理：左键wampsever可以直接看到PhpMyAdmin，点击可以直接跳转到

在这一阶段，面临的问题有，注册失败直接重定向，提示不方便。这里直接修改了规则，填写信息出现问题时，直接弹窗提示而不提交信息不重定向网页。提示直接改成弹窗提示而不是内嵌提示信息。

2.http协议

2.1 请求方式

类型 描述
GET 从服务器获取资源，如刷新网页或点击链接时使用
HEAD 类似GET但不返回实体数据，仅获取报头信息
POST 向服务器提交数据，如注册用户、登录、发表文章等
PUT 替换服务器内容（使用较少）
DELETE 请求删除指定资源（实际开发中常用GET/POST替代）
TRACE 对链路进行测试诊断
  • GET与POST区别
GET POST
用途 请求数据 发送数据
处理 从URL获取参数 从表单获取数据
可见性 参数在URL链接中可见 参数在URL链接中不可见

2.2请求头

  • Accept字段：声明客户端可接收的数据类型（如图片/json/XML/HTML等）
  • Referer字段：标识请求来源网址（用于追踪超链接跳转）
  • Accept-Language：声明客户端支持的语言类型（如中文/英文等）
  • User-Agent(UA)：
    • 包含浏览器类型和操作系统信息
    • 典型应用：移动端适配不同设备分辨率
  • Content-Type：
    • 定义传输数据的MIME类型
    • 常见类型：image/jpg, text/html, application/json等
    • 全称：Multipurpose Internet Mail Extensions

2.3 响应报文

2.3.1 响应行(Status Line)

  • 组成要素：
    • HTTP协议版本
    • 状态码
      • 1xx状态码：表示服务器已接收请求但需要继续操作
      • 2xx状态码：表示请求成功
        • 200：请求成功
        • 201：资源创建成功
        • 204：无返回内容
      • 3xx状态码：强制要求客户端访问新地址
      • 4xx状态码：明确表示客户端请求存在问题
      • 5xx状态码：服务器处理请求时发生内部错误
    • 状态描述

2.3.2 响应头(Response Header)

  • 功能：服务器返回的元数据信息
  • 常见字段：与请求头存在对应关系

2.3.3 响应体(Response Body)

  • 特点：包含实际返回内容（如HTML文档等）
  • 处理流程：浏览器接收后需进行渲染解析

2.4 HTTP常用的调试工具

  • 浏览器开发者工具：
    • 通过F12开启，使用"网络"选项卡查看HTTP详细通信
    • 可分析请求/响应头、状态码等原始数据
  • HackBar插件：
    • 浏览器插件，支持手工构造POST请求
    • 功能包括：参数注入、Referer设置、Cookie管理等
  • Burp Suite：
    • 代理拦截工具，可修改请求/响应数据包
    • 常用于安全测试

3.2 DNS信息收集

3.2.1 DNS记录类型

![[DNS记录类型.png]]

  • A记录: 最常用类型，将域名指向一个IPv4地址（如8.8.8.8）。例如购买服务器后需要将域名解析到服务器IP。

  • CNAME记录: 别名记录，将域名指向另一个域名地址。典型应用如亚马逊使用z.cn跳转到amazon.cn，谷歌使用g.cn跳转到google.cn。

  • MX记录: 邮件交换记录，用于指定邮件服务器地址。企业邮箱配置时需要在此记录中填写邮件服务商提供的地址（如腾讯企业邮的mxdomain.qq.com）。

  • TXT记录: 文本记录，常用于域名所有权验证。验证方式包括：在网站根目录上传特定文件，或在DNS解析中添加指定文本内容。

  • NS记录: 域名服务器记录，用于指定DNS解析服务商（如将域名解析交给DNSPod管理）。

  • AAAA记录: 将域名指向IPv6地址（如ff06:0:0:0:0:0:0:c3ff06:0:0:0:0:0:0:c3ff06:0:0:0:0:0:0:c3）。
